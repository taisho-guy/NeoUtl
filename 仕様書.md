# AviQtl 0.1.0 仕様書

バージョン: 0.1.0-pre
作成日: 2026-05-21
目標: プレビュー画面（Qt Quick 3D / QML シーングラフ描画）がタイムラインの操作と完全に同期して動作する状態

***

## 第1章 アーキテクチャの根本思想

AviQtlは、C++（データ正本と補間・Luaエクスプレッション計算）と QML/Qt Quick 3D/RHI（シーングラフベースの描画・エフェクト合成）を完全に分離・ネイティブ統合した**「Dual-State QML-C++ ハイブリッドアーキテクチャ」**を採用する。

### グラフィクスバックエンドの選定根拠

AviQtlは、Qt Quick 3D + QRhi ネイティブ統合を採用する。

- **Qt Quick 3D / QRhi ネイティブ統合**：Qt 6 の QRhi（Qt Rendering Hardware Interface）は Qt Quick / Qt Quick 3D のレンダリング基盤そのものであり、SwapChain 競合が構造的に発生しない。
- **マルチグラフィクスAPI対応**：Vulkan（Linuxデフォルト）、Metal（macOSデフォルト）、D3D12/D3D11（Windowsデフォルト）を、Qt Quickシーングラフのコンテキスト上で透過的に駆動する。
- **シェーダー変換**：Vulkan-style GLSLで記述したシェーダー（`.frag` / `.comp` など）は、ビルド時に `qsb` ツールによって SPIR-V / MSL / HLSL に自動変換され、クロスプラットフォームでネイティブに稼働する。

## 第2章 レイヤー定義と責務

### Layer 1: QML / Qt Quick 3D（UI・シーングラフ描画層）
- **責務**：ユーザー入力の受付、`View3D` にクリップ（`VideoObject`, `TextObject`, `RectObject` 等）を `Model` として配置した 3D レンダリング、および `ShaderEffect` + `ShaderEffectSource` チェーンによる 2D フィルタエフェクト適用。
- **接点**：`TimelineController` / `Workspace` 等のサービスプロパティを介してC++側と通信する。タイムライン更新やパラメータ変更時は、`evaluateClipParams` を呼び出してリアクティブに最新の計算済みパラメータを反映する。

### Layer 2: Bridge / DocumentModel（正本層）
- **責務**：プロジェクトの全データ（クリップ・キーフレーム・エフェクト定義）を木構造で保持する唯一の正本。Undo/Redoスタック（`QUndoStack`）、シリアライズ（`.aviqtl` ファイルの保存/読込）を担う。
- **接点**：C++側の各サービス（`ProjectService`, `TimelineService` 等）を介して状態の変更とクエリを受ける。

### Layer 3: C++ パラメータ評価・計算キャッシュ層（補助的ECS）
- **責務**：`EffectModel` に組み込まれた C++ 補間エンジンと `LuaHost` による動的エクスプレッション評価により、毎フレームのパラメータ（位置・回転・不透明度・エフェクトパラメータ等）を高速に計算して QML に提供する。
- **補助的ECS**：`ECSState` は、スレッド間（描画スレッド、オーディオスレッド、デコードスレッド）や外部システム（`AudioMixer` 等）との間で、スレッド間同期バッファ（triple-buffering）やデータ同期キャッシュとして機能する。

## 第3章 Qt Quick 3D + RHI 統合描画アーキテクチャ

### プレビュー画面のシーングラフ統合

プレビュー画面は、`QtQuick3D` の `View3D` を中核とし、タイムラインに沿って QML のクリップ定義オブジェクトを動的に配置する。

- **プレビューコンテナ**：`ui/qml/CompositeView.qml` 内で `View3D` を配置し、アスペクト比を維持する Letterboxing を行う。
- **クリップオブジェクトのインスタンス化**：QML の `Instantiator` と `NodeLoader` を用いて、`clips` モデルから対応するオブジェクト（`VideoObject.qml`, `TextObject.qml`, `RectObject.qml` 等）を動的に生成し、3D空間上の `Node` として追加・配置する。

### Videoデコードフレームの流し込み（`videoSink` 連携）

非同期デコードスレッドで FFmpeg によりデコードされた動画フレームは、C++ の `VideoFrameStore` を仲介して QML の `VideoOutput` / `videoSink` にダイレクトに流し込まれる。

```
[FFmpeg 非同期デコード]
      │
      ▼
[VideoFrameStore (C++)]
      │  registerSink() で QVideoSink を補足
      ▼
[QVideoSink (QML/C++)]
      │  videoOut.videoSink にフレームを受け渡し
      ▼
[VideoOutput (QML)]
      │  layer.enabled = true でレイヤーテクスチャ化
      ▼
[QtQuick3D Texture]
      │  sourceItem: videoOut を diffuseMap に貼り付け
      ▼
[View3D Quad Model]
```

### Compute Shader 統合（`QSGRenderNode` 方式）

エフェクトの一部で Compute Shader（SSBOの読み書きなど）を直接使用する場合、`beforeRendering` シグナルによる低レベル介入ではなく、Qt Quick シーングラフの標準的な介入アプローチである `QSGRenderNode`（`ComputeEffect` / `ComputeRenderNode`）を用いて、シーングラフ内にComputeコマンドを安全に統合する。

- `ComputeEffect` は `QQuickItem` を継承し、`updatePaintNode()` 内で `ComputeRenderNode` を生成する。
- `ComputeRenderNode` の内部で `QRhiComputePipeline` および `QRhiCommandBuffer::beginComputePass()` を実行し、SSBOバッファ（`setStorageBufferRaw()`）と統合されたゼロコピーパスを実現する。

## 第4章 フィルタエフェクト・チェーン合成

2D フィルタエフェクト（ぼかし、色調補正等）の適用は、QML の `ShaderEffect` と `ShaderEffectSource` をカスケード接続した「エフェクトチェーン」によって構築される。

### チェーン合成アルゴリズム（`ObjectRenderer.qml`）

1. クリップオブジェクトの `originalSource`（元の2Dテクスチャ等）を入力とする。
2. `Repeater` と `Loader` を用いて、有効化されているエフェクト（`BorderBlur.qml` 等）を配列順に動的にインスタンス化する。
3. 各エフェクトの `Loader` は、前段のエフェクトの出力を `inputSource` としてリアクティブに受け取る。
4. 最終段の出力を `ShaderEffectSource` でキャプチャ（テクスチャ化）し、`View3D` 内のQuadモデルの `diffuseMap` に貼り付ける。

### GLSLシェーダーの構成

各エフェクト（例: `BorderBlur.qml`）は、対応する GLSL フラグメントシェーダー（例: `borderblur.frag`）とペアで記述される。
ビルド時に `qsb` によりコンパイルされた `.qsb` ファイル（例: `borderblur.frag.qsb`）が実行時に動的に読み込まれ、グラフィクスハードウェアの性能を最大限に引き出す。

## 第5章 データツリー構造

AviQtlのプロジェクト・設定構造は以下の通り定義される。

```
SystemSettings          [永続化: user_settings.json]
└─ ProjectSettings      [永続化: *.aviqtl]
   └─ SceneSettings[]   [複数可]
      └─ Layer[]
         └─ Clip[]
            ├─ Effect[]
            │  └─ KeyframeTrack[] -> Keyframe[]
            └─ AudioPlugins[]
```

### Keyframe (キーフレーム)
パラメータ値の補間に用いるキーフレームデータ。
- `frame` (int)：クリップ開始点からの相対フレーム
- `value` (QVariant)：パラメータ値（数値、カラー文字列など）
- `interp` (QString)：補間タイプ（"linear", "ease_in_sine", "ease_out_bounce", "custom" 等）
- `points` (QVariantList)：`custom` 補間（3次ベジェ曲線）の制御点配列 `[cp1x, cp1y, cp2x, cp2y, endX, endY]`
- `modeParams` (QVariantMap)：`random` / `alternate` 補間の周期などの追加パラメータ

## 第6章 C++ 補間エンジンとエクスプレッション評価

毎フレームのパラメータ計算および補間は、C++ 側の `EffectModel` 内で高速に処理され、QML 側に `evaluatedParams()` / `evaluateClipParams()` を介して提供される。

### 補間エンジン

- **イージング関数群**：30種類以上の標準イージング関数（`linear`, `sine`, `quad`, `cubic`, `bounce`, `elastic` 等の In/Out/InOut/OutIn 派生）が C++ 内でハードコードされ、極めて低コストで計算される。
- **ベジェ数値解法（`custom`）**：`custom` 補間（3次ベジェ曲線）は、時間Xから媒介変数Tを逆算するため、ニュートン＝ラプソン法（`solveBezierT`）を用いたC++数値解法により、毎フレームの評価コストを実質ゼロに抑えつつ高精度に算出する。

### エクスプレッションの動的評価（`LuaHost`）

パラメータの静的な値が `=` から始まっている場合（例: `=time*100 + sin(time)*50`）、補間エンジンはそれを数式エクスプレッションとして扱い、`LuaHost` シングルトンを介して実行時に動的に評価する。

- `LuaHost` 内で Lua スクリプトとして高速に評価され、現在のフレームに対応する秒数（`time`）や基準値（`baseValue`）を引数として渡すことができる。

## 第7章 ECS (Entity Component System) の補助的役割

現在の AviQtl アーキテクチャにおいて、ECSはパラメータ計算や描画の主系列からは分離されているが、スレッド境界をまたぐデータ転送や、外部オーディオ処理との同期のための**補助的なメモリ同期・通信レイヤー**として機能する。

### ECSの構成と triple-buffering

- `ECSState` 内には `transforms`, `renderStates`, `audioStates`, `metadataStates` が `DenseComponentMap` で管理されている。
- C++ 側では、スレッドセーフな triple-buffering（`editIndex`, `activeIndex`, `pendingIndex` の CAS 操作）により、GUIスレッドから書き込まれた最小限のデータを、再生・音声・描画の各バックエンドスレッドに安全に引き渡す。
- `CommandSystem`（`runCommandSystem`）は、毎フレーム先頭で `CoreBridge` のリングバッファから `Seek`, `Play`, `Pause` などの再生コマンドを回収し、再生状態（`currentFrame`, `isPlaying`）に安全に反映する。

## 第8章 QML シーングラフベースの RenderGraph

自前のC++ RenderGraph DAGに代わり、QML のシーングラフにおける **`fbCaptureItem` (BaseObject.qml) のカスケード**によって RenderGraph が構築される。

### カスケード合成

- クリップが属するレイヤーの順序に従い、下位レイヤーから上位レイヤーへ向かって描画結果が重ね合わせ（ブレンド）される。
- クリップごとに `fbCaptureItem`（`_fbCaptureItemImpl`）が定義され、現在のクリップの変形（位置・回転・スケール・不透明度・ブレンドモード）を再現した2Dテクスチャを出力する。
- 複数のレイヤーやグループにまたがる変形（`GroupControl`）は、QML 側で親の変換行列（`Qt.matrix4x4`）を階層的に計算し、子の位置・回転・不透明度にリアルタイムで伝播させることで実現している。

***

## 第9章 ブレンドモードとシェーダー管理

### ブレンド合成

AviUtl互換の豊富なブレンドモードは、以下の通り実現される。

1. **基本ブレンド**：`QtQuick3D` の `DefaultMaterial` の `blendMode` プロパティ（`SourceOver`, `Screen`, `Multiply`, `ColorBurn`, `ColorDodge` 等）をダイレクトに利用する。
2. **カスタムブレンド**：QML の `ShaderEffect` または `ShaderEffectSource` に、輝度条件分岐などを記述した専用のブレンドフラグメントシェーダー（`blend_layer.frag.qsb` 等）を組み合わせることで、オーバーレイなどの高度な合成を実現する。

### シェーダー自動コンパイル

- **GLSLソース**：シェーダー（`.frag`, `.comp` など）は `Vulkan-style GLSL` で記述する。
- **CMakeビルド**：ビルド時に `Qt6::qsb` を呼び出し、各プラットフォーム向け（SPIR-V, GLSL ES, HLSL, MSL）のコードを内包した `.qsb` パッケージにコンパイルする。
- **実行時ロード**：コンパイルされた `.qsb` ファイルは、バイナリへの埋め込みを行わず、実行時に指定のディレクトリから動的にスキャン・ロードされる。

## 第10章 Bake・インスタンス化戦略

ECSへのエンティティ常駐に代わり、QML のシーングラフによる**ビューポート・時間ベースの動的インスタンス化戦略**を採用する。

- **時間ベースフィルタリング**：QML の `Instantiator` モデルにおいて、再生ヘッド（`currentFrame`）がクリップの寿命範囲（`startFrame` 〜 `startFrame + durationFrames`）にあるかどうかを監視する。
- **動的活性化/非活性化**：範囲内に入ったクリップのみを `visible = true` にしてシーングラフの描画ツリーに登録し、不要になったクリップは `visible = false` または QML Loader を使って動的にアンロードする。これにより、メモリ（VRAM/RAM）の消費を最小限に抑える。

***

## 第11章 エフェクト・オブジェクトの拡張システム

AviQtlは、C++のコードを変更・再コンパイルすることなく、JSON定義ファイルとQMLテンプレートを追加するだけで、新規エフェクトや描画オブジェクトを動的に拡張できるプラグインシステムを実装している。

- **JSONメタデータ**：エフェクトやオブジェクトのパラメータ構成、表示名、UIのコントロール（スライダー等）を定義する。
- **QMLソース**：オブジェクトの描画ロジック（`stb_image`読み込み、`Text`レンダリング、`ShaderEffect`の適用等）を記述する。
- **C++ EffectRegistry**：起動時にディレクトリを自動スキャンし、検出したプラグインを `TimelineController` / QML に動的に登録・ロードする。

## 第12章 Video / Image / Text / Audio / VideoFrameStore API の責務

### Video API
- **FFmpeg非同期デコード**：バックエンドスレッドでFFmpegを用いて動画を非同期にデコードする。
- **VideoFrameStore**：デコードされたフレームキャッシュを保持し、QML の `QVideoSink`（`videoSink`）に `registerSink` / `upload` 経由でスレッドセーフに供給する。

### Image API
- `stb_image` を用いた画像ファイルの高速デコード。
- デコード結果をQMLの `Image` または `ShaderEffectSource` に提供する。

### Text API
- Qt Quickのテキストレンダリングエンジンを活用し、QML の `Text` アイテムによりアンチエイリアスの効いた文字描画を行う。
- これをオフスクリーンテクスチャとして `View3D` 内のQuadに貼り付ける。

### Audio API (AudioMixer / Carla統合)
- **AudioMixer**：C++側の再生ヘッド（タイムライン）と完全に同期した音声ミックススレッドを駆動する。
- **Carlaプラグインホスト**：`AudioMixer` 内の音声処理チェーン（`getChain`）において、Carlaオーディオプラグイン（VST/LV2等のネイティブエフェクト）をロード・実行し、プロフェッショナルな音声効果を適用する。
