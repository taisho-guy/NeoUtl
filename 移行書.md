# AviQtl 0.1.0 移行書

仕様書: [仕様書.md](file:///home/user/ドキュメント/Projects/AviQtl/仕様書.md) 準拠
目標: プレビュー画面（Qt Quick 3D / QML シーングラフ描画）がタイムラインの操作と完全に同期して動作する「Dual-State QML-C++ ハイブリッドアーキテクチャ」への移行実績ロードマップ

***

## 概要

AviQtlは、開発の過程で当初の「C++完全低レベルECS + 自前QRhiレンダラー」という複雑な設計から、よりQtの設計思想に適合した**「Qt Quick 3D ベースのQMLシーングラフ + C++パラメータ評価・補間エンジン + QML ShaderEffect 2Dフィルタチェーン」**というハイブリッドアーキテクチャへと移行しました。

本手順書は、古い自前レンダラー設計から現在のハイブリッドアーキテクチャへの実際の移行ステップと実績を記録し、今後の開発・保守のロードマップを示すものです。

***

## Phase 3: C++ パラメータ評価・補間エンジンの構築

### Step 3-1: `EffectModel` 補間エンジンの実装
- [x] C++ 側 `EffectModel` 内に、30種類以上の標準的なイージング関数（`linear`, `sine`, `bounce`, `elastic` 等の In/Out/InOut/OutIn 派生）をハードコードで実装
- [x] 3次ベジェ曲線補間（`custom`）において、時間 X から媒介変数 T を高速かつ高精度に逆算するため、ニュートン＝ラプソン数値解法（`solveBezierT`）を C++ にて実装し、毎フレームの評価コストを実質ゼロに低減

### Step 3-2: QML リアクティブパラメータバインディング
- [x] QML 側から C++ 側の計算結果をリアクティブにフェッチするため、C++ / QML ブリッジに `evaluateClipParams` / `evaluatedParam` などのインターフェースを実装
- [x] タイムライン更新時やパラメータ変更時に、計算済みの最新パラメータを瞬時に QML の描画ツリーに反映するバインディングを確立
